# 04 Scripts & the Hook System

## 4.1 What is a Script?

### 4.1.1 Overview
A `Script` is an entity that attaches to an `Object` and intercepts its `at_*` lifecycle and interaction hooks. Scripts provide the primary mechanism for composition and reusability within Atheriz, allowing you to layer dynamic behaviors on objects without requiring complex class inheritance structures.

Common uses include timed buffs/debuffs, complex AI behaviors, localized weather effects, quest tracking, and temporary status alterations. 

Reference [`atheriz/objects/base_script.py`](../atheriz/objects/base_script.py) for the core definitions.

### 4.1.2 Script Lifecycle
1. **Creation**: Calling `Script.create(caller, name, desc)` instantiates the script and caches it in memory.
2. **Installation**: Calling `script.install()` patches the script's hooks onto the child object.
3. **Execution**: The script hooks are patched dynamically onto the parent child Object.
4. **Removal**: Calling `Script.delete()` removes the hooks from the script's child object and deletes the Script permanently.

## 4.2 Hook Decorators: `@before`, `@after`, `@replace`

### 4.2.1 How Hooks Work
Any method contained inside a Script subclass designated with the `at_` prefix acts as an interception hook upon the child Object.

Every `at_*` method inside a script **must** utilize one of the three decorators supplied in `atheriz.objects.base_script`:
- `@before`: The script's hook executes first. If it returns standard values, the original object's hook executes afterward.
- `@after`: The original object's hook executes first, followed by the script's hook execution.
- `@replace`: The original object's hook is ignored. Only the script's logic executes.

The `is_before`, `is_after`, and `is_replace` properties generated by these decorators connect directly into the object's core `@hookable` definition located in `base_obj.py`.

### 4.2.2 Practical Examples

**@before**: Creating a damage-over-time tick effect that reduces health before an object processes a tick normally.

```python
from atheriz.objects.base_script import Script, before

class PoisonScript(Script):
    @before
    def at_tick(self):
        self.child.health -= 5
```

**@after**: A diagnostics script tracking object movement passively.

```python
from atheriz.objects.base_script import Script, after

class TrackingScript(Script):
    @after
    def at_post_move(self, source, destination):
        print(f"Object {self.child.id} moved from {source} to {destination}")
```

**@replace**: A paralyze script preventing movement execution entirely.

```python
from atheriz.objects.base_script import Script, replace

class FrozenScript(Script):
    @replace
    def at_pre_move(self, destination):
        self.child.msg("You are completely frozen and cannot move.")
        return False
```

## 4.3 Creating a Custom Script

### 4.3.1 Step-by-Step: A Buff Script
A strength buff script alters an object's combat calculations for multiple ticks, eliminating itself after the duration lapses.

```python
from atheriz.objects.base_script import Script, replace

class StrengthBuff(Script):
    def at_install(self):
        self.duration = 10
        self.child.strength_modifier += 5
        self.child.msg("You feel immense strength surging through you.")

    @after
    def at_tick(self):
        self.duration -= 1
        if self.duration <= 0:
            self.child.msg("Your strength fades.")
            self.child.strength_modifier -= 5
            self.delete()
            return
```

### 4.3.2 Attaching and Removing Scripts
Assigning scripts to target objects utilizes standard API methods:
- `object.add_script(script_or_id)`
- `object.remove_script(script_or_id)`

### 4.3.3 Checking for Scripts
Checking for scripts is done using the `has_script_type()` method:
- `object.has_script_type(script_type)`

### 4.3.4 Getting Scripts by Type
Getting scripts of a specific type is done using the `get_scripts_by_type()` method:
- `object.get_scripts_by_type(script_type)`
